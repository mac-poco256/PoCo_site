When read the normally PNG file

As mentioned to read from file, the PoCo.app is assumed that a file is stored by the PoCo.app themself.

However, that limitation is too the closing world, so the PoCo.app can read the normally PNG file when a file satisfy the condition.

Please refer to the condition when be in alliance with other applications.


A file must on the condition that following:

  • The fundamental of file structure is documented in read from file.
  • A file must not contain the ilAY chunk(the image layer).
  • A file must contain within single IDAT chunk(the image data).(the separated IDAT chunk does not supported by the PoCo.app.)

Thus, when a file comprises of 256 colors and following PNG specification, file extension is renamed to ".poco", the PoCo.app can read that file which is stored by other application.
