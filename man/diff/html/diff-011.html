1.4.0 から 1.5.0 の変更点

Version 1.4.0 から Version 1.5.0 へ更新するにあたり、変更となった内容についてまとめています。


追加

新規に追加した機能は次のとおりです。

  • 出力する実行バイナリを Intel と Arm の Universal Binary としました。
  • QuickLook プレビューに対応しました。
  • ペン先・タイルパターンをデフォルトへリセットする機能を追加しました。
  • 選択中のレイヤーを真上に複製する機能を追加しました。


変更（訂正）

変更、ないし訂正した機能は次のとおりです。

  • 最新の SDK を用いてビルドすると、いくつかの View が正常に再描画されない不具合を修正しました。
  • 最新の SDK を用いてビルドすると、レイヤーウィンドウのリストに左側に余計なアキが含まれる不具合を修正しました。
  • レイヤーウィンドウのリストの各項目の幅を調整しました（省略表示になっていた不具合を修正しました）。
  • パレット詳細が正常にスクロールできない不具合を修正しました。
  • 塗りつぶしを伴ったぼかしが正常に動作しない（画像を破壊する、あるいはアプリケーションがクラッシュする）不具合を修正しました。
  • 選択枠を用いた回転・変倍をおこなったのち、その選択枠内の画像に対する編集（左右・上下反転、自動グラデーション、色置換、およびテクスチャ）が正常に動作しない（画像を破壊する、あるいはアプリケーションがクラッシュする）不具合を修正しました。
  • 自動グラデーションの詳細設定シートをキャンセルしても、シートの設定がキャンセルとならない不具合を修正しました。
  • validateMenuItem に対する実装を修正しました。この修正を実施するまでは、いくつかのメニューが不意に正常に動作しなくなっていた可能性があります。
  • ペン先・タイルパターンを UserDefaults へ保存する際に用いる archiver を NSArchiver・NSUnarchiver から NSKeyedArchiver・NSKeyunarchiver へ変更しました。
  • そのほか、deprecated と記されたメソッド・定数について修正を行いました。


削除（廃止）

変更（訂正）に記載のとおり、NSArchiver・NSUnarchiver から NSKeyArchiver・NSKeyedUnarchiver へ archiver を切り替えています。

この変更の影響で、Version 1.4.0 と Version 1.5.0 の間では前方互換性のみが維持されています。

具体的には、UserDefaults にユーザーが編集したペン先・タイルパターンがある場合、Version 1.5.0 を実行するとそれらは自動的に NSKeyedArchiver によって保存しなおされます。こののち、Version 1.4.0 やそれ以前の PoCo を起動すると、アプリケーションは正常に起動できなくなります（ペン先・タイルパターンについては保存の方法が変わったということであり、すなわち、後方互換性を維持していないということです）。

もし、Version 1.5.0 を起動したのち、Version 1.4.0 以前の PoCo へ戻る場合は、パターンをデフォルトにリセットする機能を用いるなどして、UserDefaults 内のペン先・タイルパータンをデフォルトにリセットしてください。
